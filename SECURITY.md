# 安全政策

## 负责任披露

若你发现安全漏洞，请勿在公开 Issue 中披露。可通过以下方式私下报告：

- **GitHub Security Advisories**：在本仓库的 **Security** 标签页选择 **Report a vulnerability**，按流程提交。
- 若无法使用上述方式，可在公开讨论区说明「需要私下报告安全问题」，维护者会与你联系。

我们会尽快确认并评估报告，并在修复后（在合理范围内）致谢并说明。

## 支持范围

- 本仓库内 Prizm 服务端、Electron 客户端及相关包（如 `@prizm/server`、`@prizm/client-core`、`packages/evermemos` 等）的代码与依赖。
- 不包含第三方 MCP 服务、上游 EverMemOS 等独立项目的漏洞，但若影响本仓库集成方式，也欢迎告知。

感谢你帮助 Prizm 更安全。
